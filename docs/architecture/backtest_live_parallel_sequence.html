Backtest / Paper / Live 职责链对照：统一运行时版

采用同一套职责命名，按 配置加载 -> 数据获取 -> 标准化与聚合 -> 策略执行 -> 意图到风控 -> 订单执行 -> Fill/账户/输出 的链路对齐显示。Paper 与 Live 共用运行时链路；差异只允许落在数据源、执行 adapter、 外部账户/端口、风控门禁、观测与签核状态。

已实现 部分实现 未实现 职能一致 / 共享核心

共同实现链路

三种模式共享同一条职责链：Strategy SDK -> StrategyContext -> TargetIntent -> RiskEngine -> OrderManagerActor -> ExecutionActor -> AccountActor -> reporting。 Backtest 使用 replay/simulated 边界；paper/live 使用 broker 边界。

目录原则

qts/runtime 是 backtest/live 共用运行时，不是 live 专属目录。 Paper/live 的会话编排由 RuntimeSession 持有；模式差异只落在 data source、execution adapter、runtime event sink、配置和运营门禁。

命名原则

相同职责使用同一个类名；只有边界实现带模式差异，例如 ReplayMarketDataSource / StreamingMarketDataSource、 SimulatedExecutionAdapter / BrokerExecutionAdapter。 Live capital 未签核前只允许 observation/code-path-ready。
