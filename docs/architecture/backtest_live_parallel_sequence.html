Backtest / Live 职责链对照：统一类名版

采用同一套职责命名，按 配置加载 -> 数据获取 -> 标准化与聚合 -> 策略执行 -> 意图到风控 -> 订单执行 -> Fill/账户/输出 的链路对齐显示。已实现阶段先落到目标类名，未实现阶段先占位文件和类名。

已实现 部分实现 未实现 职能一致 / 共享核心

共同目标

两条路径共享同一条职责链，差异只在边界：历史回放/实盘 feed 与 模拟执行/经纪商执行。共用核心保持一致。

目录原则

qts/runtime 是 backtest/live 共用运行时，不是 live 专属目录。 模式差异只落在 data source、execution adapter、runtime event sink。

命名原则

相同职责使用同一个类名；只有边界实现带模式差异，例如 ReplayMarketDataSource / StreamingMarketDataSource、 SimulatedExecutionAdapter / BrokerExecutionAdapter。

阶段链路图

每张卡对比 Backtest / Live，保留职责链一致性

1配置加载

职责：把外部配置转成受校验运行输入。统一目标类名： TradingRuntimeConfig。
Backtest已实现
  • BacktestRuntimeConfig（目标名）
  • 目标文件：backend/src/qts/runtime/config.py
  • 当前承接：BacktestRuntimeConfig.from_yaml、BacktestEngineConfig
Live / Paper部分实现
  • LiveRuntimeConfig（目标名）
  • 目标文件：backend/src/qts/runtime/config.py
  • 当前承接：LiveRuntimeConfig、validate_live_startup
统一输出：账户配置、风险配置、日历、adapter 边界。

2数据获取

职责：从边界源取数据并规范化。统一目标类名： MarketDataSource。
Backtest已实现
  • ReplayMarketDataSource（目标名）
  • 目标文件：backend/src/qts/data/sources/replay_market_data_source.py（占位）
  • 当前承接：ReplayMarketDataSource、HistoricalMarketDataAdapter
Live / Paper部分实现
  • StreamingMarketDataSource（目标名）
  • 目标文件：backend/src/qts/data/sources/streaming_market_data_source.py（占位）
  • 当前承接：FakeLiveFeedAdapter、IbkrMarketDataAdapter
统一约束：外部符号只允许停留在 adapter 边界。

3标准化与聚合

职责：标准化、去重、bar 聚合、时序对齐；运行时只负责把数据流接入 actor/queue。 统一目标类名：MarketDataPipeline、MarketDataFlow。
Backtest已实现
  • MarketDataPipeline（目标名）
  • 目标文件：backend/src/qts/data/market_data_pipeline.py（共享数据处理）
  • MarketDataFlow：backend/src/qts/runtime/market_data_flow.py（共享运行时编排）
  • 当前承接：BacktestActorLoop、BarAggregationPipeline
Live / Paper部分实现
  • MarketDataPipeline（目标名）
  • 目标文件：backend/src/qts/data/market_data_pipeline.py（共享数据处理）
  • MarketDataFlow：backend/src/qts/runtime/market_data_flow.py（共享运行时编排）
  • 当前承接：fake live 事件可进入 MarketDataActor，生产 loop 未闭环
统一约束：timeframe 与 session 语义由领域层定义，不可受 provider 影响。

4策略执行

职责：策略只读视图并输出目标意图。统一目标类名： StrategyExecutionPipeline。
Backtest已实现
  • StrategyExecutionPipeline（目标名）
  • 目标文件：backend/src/qts/runtime/strategy_execution_pipeline.py（已实现）
  • 当前承接：StrategyExecutionPipeline 协调 StrategyActor、SignalAggregatorActor、StrategyContext
Live / Paper部分实现
  • StrategyExecutionPipeline（目标名）
  • 目标文件：backend/src/qts/runtime/strategy_execution_pipeline.py（同上，已实现）
  • 当前承接：共享策略组件，未形成完整 live runtime 调度链
统一约束：策略侧不分模式分支，不直接触达 Order/Account/Risk。

5意图到风控

职责：目标意图解析为订单计划并执行风控。统一目标类名： TargetIntentProcessor、OrderPlanBuilder。
Backtest已实现
  • TargetIntentProcessor（目标名）
  • OrderPlanBuilder（目标名）
  • 目标文件：backend/src/qts/runtime/intent_processing.py（已实现）
  • 当前承接：OrderPlanBuilder 生成订单计划，TargetIntentProcessor 执行风控和提交订单
Live / Paper未实现
  • TargetIntentProcessor + OrderPlanBuilder（目标名）
  • 目标文件：backend/src/qts/runtime/intent_processing.py（共享处理器已实现，live runtime 未接入）
  • 缺口：live runtime 未形成 intent -> risk -> order 的完整闭环
统一约束：RiskEngine 在两种模式下都必须前置。

6订单执行

职责：订单生命周期与执行桥接。统一目标类名： ExecutionAdapter、OrderExecutionCoordinator。
Backtest已实现
  • SimulatedExecutionAdapter（目标名）
  • 目标文件：backend/src/qts/execution/adapters/simulated_execution_adapter.py（已实现）
  • 当前承接：SimulatedExecutionAdapter、ExecutionActor
Live / Paper部分实现
  • BrokerExecutionAdapter（目标名）
  • 目标文件：backend/src/qts/execution/adapters/broker_execution_adapter.py（已实现基础 broker adapter）
  • 当前承接：BrokerExecutionAdapter 可接入 ExecutionActor；IbkrOrderExecutionAdapter 仍是 provider shape 映射
统一约束：OrderManagerOwner 与 ExecutionActor 的边界不变。

7Fill / 账户 / 输出

职责：fill 回流、账户更新与运行输出。统一目标类名： ExecutionReportHandler、RuntimeEventSink、ReportWriter。
Backtest已实现
  • ExecutionReportHandler（目标名）
  • 目标文件：backend/src/qts/runtime/execution_report_handler.py（已实现）
  • BacktestRuntimeEventSink：backend/src/qts/runtime/sinks/backtest.py（已实现）
  • BacktestReportWriter：backend/src/qts/reporting/backtest.py（已实现）
  • 当前承接：ExecutionReportHandler 处理 ExecutionReport -> ApplyFill -> AccountActor
Live / Paper部分实现
  • ExecutionReportHandler、LiveRuntimeEventSink（目标名）
  • LiveRuntimeEventSink：backend/src/qts/runtime/sinks/live.py（占位）
  • LiveReportWriter：backend/src/qts/reporting/live.py（占位）
  • 当前承接：broker callback 可经 BrokerExecutionAdapter 标准化后接入 ExecutionReportHandler
  • 缺口：live runtime 事件输出和 live report writer 完整链路
统一约束：账户状态变更只允许来自标准化 fill 回报并经 AccountActor。
推荐目录规范：共享运行时 + 模式边界
