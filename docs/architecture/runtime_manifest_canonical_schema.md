# Runtime Manifest Canonical Schema

`qts.reporting.base.RuntimeManifest` owns the shared manifest schema for
backtest, paper, and live runtime artifacts. Report writers may add
mode-specific evidence, but every manifest must include these canonical fields:

- `run_id`
- `runtime_instance_id`
- `runtime_mode`
- `market_data_environment`
- `execution_environment`
- `account_environment`
- `live_order_permission`
- `config_hash`
- `topology_hash`
- `startup_checklist_hash`
- `event_schema_version`
- `artifact_schema_version`
- `created_at`
- `source_commit`
- `operator_identity_hash`

`manifest_hash` is deterministic and computed by
`RuntimeManifest.hash_payload(...)` over the full JSON payload excluding the
`manifest_hash` field itself. `RuntimeManifestRecord.load(path)` is the
reporting-owned query boundary for loading a manifest artifact and querying
top-level fields or the canonical `manifest_hash`.

Backtest manifests use explicit non-broker sentinel values generated by
`BacktestArtifactWriter` for broker-only facts:

- `source_commit`: `not-applicable-backtest`
- `operator_identity_hash`: `sha256:not-applicable-backtest`
- `startup_checklist_hash`: `sha256:not-applicable-backtest`

Broker runtime manifests must provide `runtime_instance_id`, `source_commit`,
and `operator_identity_hash` to the writer. Missing canonical fields fail
manifest validation before the artifact is written.
