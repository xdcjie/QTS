Boundary Answer

现在是不是所有模块都通过 Actor 通信？

不是。Actor 是运行时状态和消息边界；配置解析、历史数据读取、symbol/roll resolution、risk check、DataView 和 report writer 仍是同步服务或值对象。

Actor 主干

MarketDataActor、StrategyActor、SignalAggregatorActor、 OrderManagerActor、ExecutionActor、AccountActor 是运行时消息边界。

Backtest 特点

Backtest 由 BacktestEngine 同步驱动 Actor：它读取 replay 事件，向 ActorRef 发消息，再调用 process_all() 推进 mailbox。

Live 特点

Live 的 market data 从 adapter / feed 进入 MarketDataActor；进入 MarketDataActor 之后，和 backtest 的策略侧数据路径一致。

Mode Entry

Backtest 和 Live 怎样进入 Actor 主干

BACKTEST 历史行情 Feed 入口

  • BacktestRunConfig.from_yaml 解析 run config。
  • HistoricalDataConfig + HistoricalCatalog 定位 CSV / chain。
  • HistoricalBarStream 作为 historical feed 产出 source Bar，连续合约记录 FutureRollSelection。
  • BacktestEngine 把 source bar 包成 MarketDataEvent，同步投递给 MarketDataActor。
  • 从 MarketDataActor 发布出的 completed bar 开始，策略路径和 live 对齐。

LIVE 实时行情 Feed 入口

  • LiveFeedAdapter 或 broker market data adapter 对接外部行情。
  • SubscribeMarketData 由策略/运行时声明逻辑订阅。
  • MarketDataActor 规划 physical subscription，接收规范化 payload。
  • 如果 source timeframe 与 requested timeframe 不同，MarketDataActor 内部聚合。
  • 发布给订阅者的 payload 是策略侧统一数据产品。

Message Contracts

每个 Actor 的输入和输出

ACTOR MarketDataActor

  • 输入：SubscribeMarketData、MarketDataEvent
  • 输出：Bar、Quote、Tick
  • 职责：订阅规划、fan-out、bar aggregation、completed bar gating。

ACTOR StrategyActor

  • 输入：StrategyBarEvent、StrategyFinalize
  • 输出：StrategyBarResult、StrategyFinalized
  • 职责：更新 ctx.data、ctx.portfolio、indicator，然后调用策略。

ACTOR SignalAggregatorActor

  • 输入：StrategySignalEvent
  • 输出：AggregatedSignalBatch
  • 职责：策略 intent 到订单流前的聚合边界。

ORDER OrderManagerActor

  • 输入：SubmitOrder、ExecutionReport
  • 输出：OrderExecutionRequest、ApplyFill
  • 职责：订单状态机、成交归档、将 fill 发给账户。

EXEC ExecutionActor

  • 输入：OrderExecutionRequest
  • 输出：ExecutionReport
  • 职责：调用 backtest simulator、paper adapter 或 live broker adapter。

STATE AccountActor

  • 输入：ApplyFill
  • 输出：AccountSnapshot 只读查询
  • 职责：唯一持有 cash、positions 和 fill idempotency。

What Is Not An Actor

同步服务和 Actor 的关系

模块 / 类 是不是 Actor 作用 接入位置
BacktestRunConfig 否 解析 backtest run identity、时间范围、策略配置、market data reference。 运行前配置阶段。
HistoricalCatalog / HistoricalBarStream 否 定位历史数据文件，读取 CSV，转换为 Bar，处理历史链和 roll selections。 Backtest feed adapter 阶段，进入 MarketDataActor 之前。
FutureRollRegistry / InstrumentRegistry 否 symbol、continuous future、concrete contract 和 execution price resolution。 StrategyContext 和 order intent 处理阶段同步调用。
RiskEngine 否 对 order request 做同步 risk decision。 OrderManagerActor 收到 SubmitOrder 之前。
HistoricalDataPortal / DataView 否 构造策略在当前 as_of 可见的数据切片。 StrategyBarEvent 的 payload。
StreamingBacktestArtifactWriter 否 写 orders、fills、trade ledger、equity curve、manifest 和 summary。 Backtest output 阶段。
