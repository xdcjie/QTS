QTS Project Panorama

以 Actor 为边界的运行时全景图

QTS 的配置解析、历史 replay input 构造、registry、risk、reporting 不是 Actor；真正的运行时通信主干从 MarketDataActor 开始，通过 ActorRef 和 Mailbox 串起策略、信号、订单、执行和账户状态。

Source / Adapter：数据源、配置、CSV、live feed
Actor：通过消息通信的运行时边界
Service / Registry：同步调用的领域服务
Order / Execution：订单执行链路
State / Output：账户状态和报告输出

Boundary Answer

现在是不是所有模块都通过 Actor 通信？

不是。Actor 是运行时状态和消息边界；配置解析、历史数据读取、symbol/roll resolution、risk check、DataView 和 report writer 仍是同步服务或值对象。

Actor 主干

MarketDataActor、StrategyActor、SignalAggregatorActor、 OrderManagerActor、ExecutionActor、AccountActor 是运行时消息边界。

Backtest 特点

HistoricalCatalog 和 BacktestInputBuilder 先构造 replay-ready inputs；BacktestEngine 只消费这些 inputs 并同步驱动 Actor mailbox。

Live 特点

Live 的 market data 从 adapter / feed 进入 MarketDataActor；进入 MarketDataActor 之后，和 backtest 的策略侧数据路径一致。

Mode Entry

Backtest 和 Live 怎样进入 Actor 主干

BACKTEST 历史行情 Feed 入口

  • BacktestRunConfig.from_yaml 解析 run config。
  • HistoricalCatalogLoadConfig 把 legacy root 或 project historical config 归一成一个 catalog construction input。
  • HistoricalCatalog.load 定位 CSV / chain，并由 catalog/config 类自己拥有解析、归一化和路径选择规则。
  • BacktestInputBuilder 消费 catalog，产出 replay bars、dataset metadata、instrument registry、contract multipliers 和 roll registry。
  • BacktestEngine.from_config 消费准备好的 inputs；它不再读取 historical config 或 chain 文件来补做 input assembly。
  • BacktestEngine 把 source bar 包成 MarketDataEvent，同步投递给 MarketDataActor。
  • 从 MarketDataActor 发布出的 completed bar 开始，策略路径和 live 对齐。

LIVE 实时行情 Feed 入口

  • LiveFeedAdapter 或 broker market data adapter 对接外部行情。
  • SubscribeMarketData 由策略/运行时声明逻辑订阅。
  • MarketDataActor 规划 physical subscription，接收规范化 payload。
  • 如果 source timeframe 与 requested timeframe 不同，MarketDataActor 内部聚合。
  • 发布给订阅者的 payload 是策略侧统一数据产品。

Runtime Flow Detail

运行链路阶段详解

下面按运行顺序展开。每个阶段都说明输入、Owner / 边界、处理动作、输出、必须守住的规则和验证依据， 让读者可以从 Backtest 或 Live 一直追踪到订单、成交、账户和输出。

01SOURCE

配置与输入装配

SOURCE

Backtest 先把 run config、historical config、CSV / chain、roll selection 装配成 replay-ready inputs；Live / paper 先把策略侧 logical subscription 映射到 provider 支持的 physical subscription。

输入BacktestRunConfig、historical catalog、live subscription request。
Owner / 边界配置 loader、HistoricalCatalog、BacktestInputBuilder、feed adapter。
处理动作解析路径、归一化 source、构造 registry / roll registry、规划订阅。
输出Backtest replay stream 或 live source subscription，准备生成内部 market data payload。
必须守住broker / data-source symbol 只能停留在 adapter 边界；core 使用 InstrumentId。
验证依据docs/architecture/backtest_live_parity.md；historical config / input builder unit tests。

Backtest 差异：历史数据、roll selections、dataset metadata 在进入 engine 前完成装配。

Live 差异：外部 provider capability 只影响 physical subscription，不改变策略请求的 bar 语义。

02INGEST

行情归一化进入 MarketDataActor

ACTOR

Historical replay、paper feed、live adapter 都把外部 payload 转换为内部 Bar、 Quote、Tick 或封装后的 MarketDataEvent，再投递给 MarketDataActor。

输入CSV bar、IBKR tick/quote/bar、fake feed event、replay clock event。
Owner / 边界feed adapter 负责外部格式归一化；MarketDataActor 是 runtime 入口。
处理动作建立 actor-facing payload，按 source / instrument / timeframe 投递。
输出MarketDataActor mailbox 中的 market data message。
必须守住市场数据不经过 ExecutionActor；订单执行不经过 market data adapter。
验证依据docs/architecture/runtime_flow.md；IBKR/fake feed integration tests。

汇合点：进入 MarketDataActor 后，Backtest / paper / live 开始共享策略侧数据路径。

边界：adapter 可处理外部协议，但不能拥有 core runtime 状态。

03BAR GATE

聚合、completed bar gating 与 fan-out

BAR

MarketDataActor 根据 logical subscription 和 source timeframe 决定是否聚合， 并只把 finalized / completed bars 发送给下游订阅者。

输入source bar / tick / quote 和策略声明的 logical timeframe。
Owner / 边界MarketDataActor 拥有 per stream BarAggregator state。
处理动作按 (instrument_id, timeframe, session_id) 更新 aggregation state，完成 fan-out。
输出AggregationResult.completed 中的 bar、quote、tick。
必须守住AggregationState 不暴露给 StrategyActor、DataView 或策略代码。
验证依据docs/testing/domain_invariants.md；bar aggregation anchor tests。

时间规则：intraday bars 按交易所时间 clock-aligned；1d bars 按 session-aligned。

可见性：下游只能看到 completed data，不能看到正在聚合的 partial bucket。

04STRATEGY

StrategyActor、StrategyContext、DataView

ACTOR

StrategyActor 接收 finalized market data，刷新 StrategyContext、 DataView、indicator / factor facade，然后调用用户策略。

输入StrategyBarEvent、completed bar、只读 portfolio snapshot。
Owner / 边界StrategyActor 拥有策略实例；SDK 只暴露用户 API。
处理动作构造 as-of 可见数据切片，调用 strategy.on_bar。
输出TargetIntent 或空结果，封装为策略输出事件。
必须守住策略不能直接访问 Actor、Broker、RiskEngine、OrderManager 或 broker mapping。
验证依据docs/strategy_sdk/strategy_api.md；Strategy SDK and DataView unit tests。

Backtest 安全：DataView 必须 time-sliced，不允许 future data 泄漏。

Mode parity：策略 API 不因 Backtest / paper / live 改变。

05INTENT

TargetIntent 聚合与账户快照

SIGNAL

策略输出的目标仓位意图先进入聚合边界，再结合只读账户/组合状态转换为可检查的 order request。

输入StrategyBarResult、TargetIntent、AccountSnapshot。
Owner / 边界SignalAggregatorActor 负责策略 intent 到订单流前的聚合。
处理动作合并同一策略/账户/资产的 intents，解析 instrument / roll / target delta。
输出AggregatedSignalBatch 或准备提交 risk check 的 order intent。
必须守住intent 不是订单；账户状态只读，不能由策略或 signal 聚合直接修改。
验证依据docs/architecture/backtest_live_parity.md；target emitter / asset resolver tests。

Resolution：continuous futures 必须解析到具体可交易合约后才能形成订单。

State：账户 cash / positions 仍由 AccountActor 唯一拥有。

06RISK

RiskEngine 到 OrderManagerActor

ORDER

所有 order request 都必须先经过 RiskEngine 的显式 decision；只有 accepted request 才能提交给 OrderManagerActor。

输入order intent / order request、账户快照、market context、risk config。
Owner / 边界RiskEngine 做同步 decision；OrderManagerActor 拥有订单状态。
处理动作执行 max qty / notional / session / kill-switch 等规则，创建或拒绝订单。
输出RiskDecision、SubmitOrder、订单状态机事件。
必须守住risk checks 不能被 SDK、API、Backtest 或 Live 路径绕过。
验证依据docs/architecture/review_checklist.md；risk and order state tests。

拒绝路径：risk rejection 必须可审计，不能静默丢弃。

订单状态：订单 lifecycle 只由 OrderManagerActor 推进。

07EXEC

ExecutionActor、ExecutionReport、AccountActor

STATE

OrderManagerActor 只把执行请求发给 ExecutionActor；broker / simulator 返回的结果先规范化为 ExecutionReport，再回到订单状态机和账户 actor。

输入OrderExecutionRequest、broker / simulator response、ExecutionReport。
Owner / 边界ExecutionActor 调 adapter；AccountActor 唯一修改 cash / positions。
处理动作提交、取消、替换、规范化回报、应用 fill、更新订单和账户。
输出ExecutionReport、ApplyFill、AccountSnapshot。
必须守住broker callbacks 不能直接改账户；fills 必须 idempotent。
验证依据docs/runtime/live_recovery_source_of_truth.md；execution / fill idempotency tests。

Adapter 差异：Backtest 使用 simulator；paper / live 使用 broker adapter，但内部 report contract 相同。

账户状态：cash / positions 只能从 validated fill path 变化。

08OUTPUT

Artifacts / API / WebSocket 输出

OUTPUT

Backtest 生成可复核 artifacts；API 和 WebSocket 面向前端或外部调用方发布状态、订单、风险、 账户和运行状态，不泄漏 actor internals。

输入订单事件、fills、账户快照、equity curve、runtime status。
Owner / 边界artifact writer、application services、API schemas、WebSocket DTOs。
处理动作写 orders / fills / ledger / summary，映射 DTO，发布状态流。
输出Backtest report files、REST responses、WebSocket updates、observability events。
必须守住Frontend / API 不实现交易逻辑，不直接操作 actor 内部状态。
验证依据docs/architecture/module_boundaries.md；API and report unit/integration tests。

可复核：Backtest artifacts 必须能追溯订单、成交、账户和 dataset metadata。

外部接口：API schemas 是边界对象，不暴露 mailbox、actor ref 或内部订单状态机对象。

Message Contracts

每个 Actor 的输入和输出

ACTOR MarketDataActor

  • 输入：SubscribeMarketData、MarketDataEvent
  • 输出：Bar、Quote、Tick
  • 职责：订阅规划、fan-out、bar aggregation、completed bar gating。

ACTOR StrategyActor

  • 输入：StrategyBarEvent、StrategyFinalize
  • 输出：StrategyBarResult、StrategyFinalized
  • 职责：更新 ctx.data、ctx.portfolio、indicator，然后调用策略。

ACTOR SignalAggregatorActor

  • 输入：StrategySignalEvent
  • 输出：AggregatedSignalBatch
  • 职责：策略 intent 到订单流前的聚合边界。

ORDER OrderManagerActor

  • 输入：SubmitOrder、ExecutionReport
  • 输出：OrderExecutionRequest、ApplyFill
  • 职责：订单状态机、成交归档、将 fill 发给账户。

EXEC ExecutionActor

  • 输入：OrderExecutionRequest
  • 输出：ExecutionReport
  • 职责：调用 backtest simulator、paper adapter 或 live broker adapter。

STATE AccountActor

  • 输入：ApplyFill
  • 输出：AccountSnapshot 只读查询
  • 职责：唯一持有 cash、positions 和 fill idempotency。

What Is Not An Actor

同步服务和 Actor 的关系

模块 / 类 是不是 Actor 作用 接入位置
BacktestRunConfig 否 解析 backtest run identity、时间范围、策略配置、market data reference。 运行前配置阶段。
HistoricalDataConfig / HistoricalCatalog 否 拥有 historical data YAML 解析、root 归一化、CSV / chain 路径解析和 catalog construction。 Backtest feed adapter 的 catalog 边界，进入 input builder 之前。
BacktestInputBuilder / HistoricalBarStream 否 读取 CSV，过滤请求 symbols，处理 continuous future roll selections，并输出 replay-ready input bundle。 Backtest feed adapter 阶段，进入 BacktestEngine 之前。
FutureRollRegistry / InstrumentRegistry 否 symbol、continuous future、concrete contract 和 execution price resolution。 StrategyContext 和 order intent 处理阶段同步调用。
RiskEngine 否 对 order request 做同步 risk decision。 OrderManagerActor 收到 SubmitOrder 之前。
HistoricalDataPortal / DataView 否 构造策略在当前 as_of 可见的数据切片。 StrategyBarEvent 的 payload。
StreamingBacktestArtifactWriter 否 写 orders、fills、trade ledger、equity curve、manifest 和 summary。 Backtest output 阶段。
